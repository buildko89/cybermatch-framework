# CyberMatch Roadmap

## Phase1 Summary

Phase1 established defense effectiveness evaluation.

Main outcomes:

- Critical asset protection
- Neutralization score
- Post-decoy compromise analysis
- Defense objective score
- MTD cost and action metrics

## Phase2 Summary

Phase2 introduced attacker cognition and Decision Neutralization.

Main outcomes:

- Perceived utility
- Confidence
- Frustration
- AI decision cost
- Cognitive Neutralization Score
- CNS objective score

## Phase3 Summary

Phase3 validated Decision Neutralization against stronger attackers.

Main outcomes:

- Adaptive attacker
- Preference attacker
- Path attacker
- Planning attacker
- Trust-aware attacker
- Expected utility attacker

## Phase4 Summary

Phase4 moved CyberMatch from adaptive defense to intelligence-driven active defense.

Main outcomes:

- Adaptive Defender
- Mission Belief
- State Belief
- Virtual Enterprise Topology
- Critical Path Intelligence
- Intelligence Decision Matrix
- Defense Campaign
- Mission Mutation
- Intent Deception
- Noise Injection
- Adversarial Signal

## Phase5 Summary

Phase5 introduced attacker-network and counter-deception co-evolution.

Main outcomes:

- Phase5.1 Multi-Attacker Coalition
- Phase5.2 Coordination Cost, Information Loss, Trust Degradation
- Phase5.2b Test Runtime Optimization
- Phase5.3 Counter-Deception Defender
- Phase5.4 Counter-Deception Aware Attacker
- Phase5.5 Counter-Deception Hunting

## Phase6 Candidates

Phase6 should focus on turning CyberMatch into a reusable evaluation platform.

Candidate work:

- Product Plugin Interface
- Scenario Import
- Product Evaluation Matrix implementation
- Reusable enterprise topology library
- Report templates for product comparison

Recommended Phase6 priority:

```text
Product Plugin Interface
Scenario Import
Enterprise Topology Library
```

## Phase7 Candidates

Phase7 should focus on realism and operational adoption.

Candidate work:

- Human Behavior Layer
- SOC workflow modeling
- Analyst trust and fatigue
- Trust Network Layer for attacker coalitions
- ATT&CK mapping and replay bridge
- External telemetry adapter

## v1.0 Candidate

CyberMatch v1.0 should be declared when the framework can support repeatable product and strategy evaluation across:

- Multi-mission attackers
- Adaptive defenders
- Coalition attackers
- Counter-deception defenders
- Aware and hunting attackers
- Enterprise topology templates
- Product plugin models
- Scenario import
- Reproducible reports
- Fast validation profiles

## Roadmap Principle

The next phases should avoid adding attackers or defenders for their own sake. The priority should shift toward:

```text
research simulator
to
evaluation platform
to
product decision tool
```
